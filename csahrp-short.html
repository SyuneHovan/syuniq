COMPETITION ON STRINGS IN C#, ARE THEY COMPARED AS A REFERENCE OR VALUE TYPE?

Value Comparison: In C#, strings are compared by value using == and .Equals(), which check their contents, not memory locations.

String Interning: Different variables may reference the same memory location due to string interning, but this doesn't affect value comparison.

Reference Equality: To check if two strings are the same instance in memory, use Object.ReferenceEquals().

HOW IS THE SYNCHRONIZATION DONE IN C#?

Definition: A synchronization context is an abstraction that provides a way to manage the execution of code across different threads. It dictates how and where certain operations (like updates to the UI) should be executed.

Purpose: It helps maintain thread safety by ensuring that certain actions are performed on the correct thread, especially when dealing with UI components in applications.

CONCURRENCY, ASYNCHRONISM, PARALLELISM, and MULTI-THREADING IN C#?

1. Concurrency

  • Definition: Managing multiple tasks at overlapping times, not necessarily simultaneously.

  • Use Cases: Useful for I/O operations to keep applications responsive.

  • Implementation: Achieved via async/await, Task, or Thread.

2. Asynchronism

  • Definition: Running tasks in the background without blocking the main thread.

  • Use Cases: Keeps UI applications responsive during long operations.

  • Implementation: Use async and await keywords.

3. Parallelism

  • Definition: Executing multiple tasks simultaneously on multiple processors.

  • Use Cases: Ideal for CPU-bound tasks to reduce execution time.

  • Implementation: Use the Parallel class.

4. Multi-threading

  • Definition: Running multiple threads within a single process.

  • Use Cases: Efficient for handling concurrent operations like user requests.

  • Implementation: Create threads using the Thread class.

Key Differences

  • Concurrency: Overlapping execution (I/O-bound).

  • Asynchronism: Non-blocking background tasks (UI responsiveness).

  • Parallelism: True simultaneous execution (CPU-bound).

  • Multi-threading: Concurrent thread management (background tasks).

WHAT IS ENCAPSULATION IN C#?

Encapsulation in C# is the practice of bundling data (fields) and methods (functions) within a class, restricting direct access to the object's components. It protects data integrity by preventing unauthorized modifications.

Example:

In a BankAccount class:

  • Private field: balance

  • Public methods: Deposit() and GetBalance()

Importance:

  • Data Protection: Safeguards against unintended changes.

  • Improved Maintenance: Allows internal changes without affecting external code.

  • Clear Interface: Simplifies interaction with the class, hiding internal complexities.

WHAT IS POLYMORPHISM IN C#?

Types:

  1. Compile-Time (Method Overloading): Same method name, different parameters.

  2. Runtime (Method Overriding): Base class methods overridden in derived classes.

Example:

  • Animal Class: Has a Speak method.

  • Dog Class: Implements Speak to say "BOWW."

  • Cat Class: Implements Speak to say "MEW."

  • Usage: An array of Animal can call Speak, returning different results based on the actual object type.

Summary:

Polymorphism enables flexible interaction with various object types through a common interface, promoting cleaner and more extensible designs.
